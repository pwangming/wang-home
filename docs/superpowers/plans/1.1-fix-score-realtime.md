## 🐛 Bug 描述

游戏进行中，右侧 sidebar "当前得分"显示 0，不会随吃食物而增加。只有在游戏结束后才能看到最终分数。

## 🔍 根因

| 位置 | 问题 |
|---|---|
| `client/src/components/game/SnakeGame.vue:52` | `const score = ref(0)` 是子组件内部 state |
| `client/src/components/game/SnakeGame.vue:220` | `score.value += ...` 只在子组件内累加 |
| `client/src/components/game/SnakeGame.vue:242` | 只在 `gameOver` 时 emit，中间过程不 emit |
| `client/src/composables/useGameSession.js:13` | 父级 `currentScore = ref(0)` |
| `client/src/composables/useGameSession.js:50` | 只在 `handleGameOver` 时更新 |

**结论：** 子组件 score 变化从未实时传给父级，导致 sidebar 的 `:score="currentScore"` 一直是 0。

## 💡 修复方案（方案 A）

SnakeGame 通过 `watch(score)` emit `scoreUpdate` 事件 → GameView 接收 → 调 useGameSession 新增的 `updateScore(n)` 方法 → `currentScore` 实时更新。

### 改动清单

**1. `client/src/components/game/SnakeGame.vue`**

- `defineEmits(['gameOver', 'eatFood'])` → 加 `'scoreUpdate'`
- 新增 watch：
  ```js
  watch(score, (v) => emit('scoreUpdate', v))
  ```

**2. `client/src/composables/useGameSession.js`**

- 新增函数：
  ```js
  function updateScore(n) {
    currentScore.value = n
  }
  ```
- `return { ... }` 中加入 `updateScore`

**3. `client/src/views/GameView.vue`**

- 从 useGameSession 解构时加 `updateScore`
- `<SnakeGame>` 加属性 `@score-update="updateScore"`

## ⚠️ 风险与缓解

| 风险 | 缓解 |
|---|---|
| watch 可能高频触发 | score 只在吃食物时变，频率极低 |

| 响应式循环 | 单向 emit，父不回写子 score，无循环 |
