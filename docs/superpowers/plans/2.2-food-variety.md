## 📍 现状摘要

当前游戏只有一种食物（`client/src/components/game/SnakeGame.vue:88-98`）：

- 随机位置
- 每吃一次 `score += 1 * scoreMultiplier`

> 注：2.0 会先把事件签名统一到新契约。2.2 在此基础上扩展多食物与 buff，不再承担“签名迁移”工作。

## 🎯 食物类型（已确认）

5 种食物、概率加权：

| 类型 | 颜色 | 出现概率 | 得分 | 特殊效果 |
|---|---|---|---|---|
| 🟢 **普通** | 绿色 | 70% | +1 | 无 |
| 🟡 **金币** | 金色 | 15% | +5 | 无 |
| 🔵 **速度药水** | 蓝色 | 8% | +1 | 5 秒内速度 ×0.6（变慢，更易操作） |
| 🟣 **幽灵模式** | 紫色 | 5% | +1 | 3 秒内穿墙+穿自身 |
| 💎 **钻石**（罕见） | 白/彩虹 | 2% | +20 | 无 |

**得分会乘以当前 scoreMultiplier（速度挡加成）保持现有逻辑。**

### 设计原则

1. **不改变游戏节奏** — 默认 70% 普通食物保证体验一致
2. **增益型特效**（慢速、无敌）而非**减益型**（变快、缩短）— 避免挫败感
3. **每局最多 3 个特殊食物并发**（防屏幕花）
4. **已存在的食物不受新刷新影响**（避免突然消失打击用户）

## 💡 方案设计（决策已确认）

### 决策 1：特效时间与 Buff 系统架构 ✅ A 瞬时 buff

- 吃到后立刻生效，`setTimeout` 结束
- 理由：简单，5 种食物够用。未来多特效叠加再重构为 Buff 管理器

### 决策 2：视觉表现 ✅ CSS/Canvas 简单差异化

- 普通：绿色圆圈（现有样式保留）
- 金币：金色闪烁（`animation: pulse`）
- 速度药水：蓝色 + 慢速粒子
- 幽灵模式：紫色 + 半透明闪烁
- 钻石：彩虹色 + 旋转

**加形状差异**（圆/方/星）防色盲，不仅靠颜色。

### 决策 3：和成就系统联动？ ✅ 本计划不联动

- 吃到特殊食物时 emit `eatFood({ type, score })` 事件
- 迭代 3 成就系统回来订阅

### 决策 4：倍率叠加规则 ✅ 相乘

- 药水是临时 buff，**乘在现有 speedMultiplier 上**（不覆盖）
- 例：速度挡 1.5x × 药水 ×0.6 = 真实速度 0.9x

### 决策 5：scoreMultiplier 和特殊食物得分 ✅ 受影响

- 金币 `score += 5 * scoreMultiplier`
- 钻石 `score += 20 * scoreMultiplier`
- 理由：保持一致性

### 决策 6：音效 ✅ 合成差异化，复用 useSound

**复用已有基建**：
- `client/src/composables/useSound.js` — 已有 `soundEnabled` ref + localStorage + `toggle` + `playEat`
- `GameView.vue` 已有 🔔/🔕 按钮接 `soundToggle`
- 现 `playEat()` 只有一种合成音（square 440→880 chirp）

**改造**：扩展 `useSound` 加 `playSound(type)`，根据食物类型合成不同音：

| 食物 | 合成参数 |
|---|---|
| 普通 | 现有 440→880 chirp（square，~150ms） |
| 金币 | 双音叮叮：880→1320→1760，sine，~200ms |
| 速度药水 | 低频气泡：200→150 下降，sine，~300ms |
| 幽灵 | whoosh：噪声 + 低通滤波，或低频扫频 1000→200，~250ms |
| 钻石 | sparkle：多振荡器和弦（C5+E5+G5），triangle，~400ms |

**不引入 mp3**，零资源、零许可问题。以后想换 mp3 再重构 composable。

**兼容性**：保留 `playEat()` 原签名（= `playSound('normal')` 别名），避免调用方迁移。

### 0. 扩展 `client/src/composables/useSound.js`（复用现有）

在现有 composable 基础上新增 `playSound(type)`：

```js
// 保留 playEat 兼容；新增分派器
function playSound(type) {
  if (!soundEnabled.value) return
  switch (type) {
    case 'coin':    return playCoin()
    case 'slow':    return playSlow()
    case 'ghost':   return playGhost()
    case 'diamond': return playDiamond()
    case 'normal':
    default:        return playEat()  // 复用原逻辑
  }
}

// 示例：金币双音叮叮
function playCoin() {
  const ctx = getAudioContext()
  if (ctx.state === 'suspended') ctx.resume()
  const freqs = [880, 1320, 1760]
  freqs.forEach((f, i) => {
    const osc = ctx.createOscillator()
    const gain = ctx.createGain()
    osc.type = 'sine'
    osc.frequency.setValueAtTime(f, ctx.currentTime + i * 0.06)
    gain.gain.setValueAtTime(0.10, ctx.currentTime + i * 0.06)
    gain.gain.exponentialRampToValueAtTime(0.001, ctx.currentTime + i * 0.06 + 0.10)
    osc.connect(gain); gain.connect(ctx.destination)
    osc.start(ctx.currentTime + i * 0.06)
    osc.stop(ctx.currentTime + i * 0.06 + 0.10)
  })
}
// playSlow / playGhost / playDiamond 类似模板
```

### 1. 新增 `client/src/lib/food.js`（食物定义）

```js
export const FOOD_TYPES = {
  NORMAL: { id: 'normal', color: '#4ade80', score: 1, weight: 70, effect: null },
  COIN:   { id: 'coin',   color: '#fbbf24', score: 5, weight: 15, effect: null },
  SLOW:   { id: 'slow',   color: '#3b82f6', score: 1, weight: 8,
            effect: { type: 'speed', multiplier: 0.6, durationMs: 5000 } },
  GHOST:  { id: 'ghost',  color: '#a855f7', score: 1, weight: 5,
            effect: { type: 'ghost', durationMs: 3000 } },
  DIAMOND:{ id: 'diamond',color: '#e0e7ff', score: 20, weight: 2, effect: null }
}

// 加权随机选择
export function pickFoodType() {
  const total = Object.values(FOOD_TYPES).reduce((s, f) => s + f.weight, 0)
  let r = Math.random() * total
  for (const food of Object.values(FOOD_TYPES)) {
    if (r < food.weight) return food
    r -= food.weight
  }
  return FOOD_TYPES.NORMAL
}
```

const activeBuffTimers = ref([])

function handleEatFood() {
  const { id, score: addScore, effect } = food.value.type
  const gained = Math.round(addScore * props.scoreMultiplier)
  score.value += gained

  // emit 对象化 payload（版本化，方便未来扩展不改签名）
  emit('eatFood', { type: id, score: gained })
  emit('playSound', id)

  if (effect) applyEffect(effect)
  placeFood()
}

function applyEffect(effect) {
  if (effect.type === 'speed') {
    activeSpeedBuff.value = effect.multiplier
    const t = setTimeout(() => { activeSpeedBuff.value = 1 }, effect.durationMs)
    activeBuffTimers.value.push(t)
  } else if (effect.type === 'ghost') {
    isGhost.value = true
    const t = setTimeout(() => { isGhost.value = false }, effect.durationMs)
    activeBuffTimers.value.push(t)
  }
}

function clearAllBuffTimers() {
  activeBuffTimers.value.forEach(clearTimeout)
  activeBuffTimers.value = []
  activeSpeedBuff.value = 1
  isGhost.value = false
}

// 在 handleGameOver 和 onUnmounted 内调 clearAllBuffTimers()

const gameSpeed = computed(() =>

## ⚠️ 风险与缓解

| 风险 | 缓解 |
|---|---|
| setTimeout 在 game over / unmount 时未清理 | 在 `handleGameOver` 和 `onUnmounted` 清所有 buff timer |
| `activeSpeedBuff` 改变触发 `gameSpeed` computed，可能让 `setInterval` 不重新调度 | 现有代码第 296 行已 watch `speedMultiplier` 重启 interval；同样的 watch 需要监听 `activeSpeedBuff` |
| 特殊食物概率偏差大 | 权重测试验证分布 |
| 色盲不友好 | 加形状差异（圆/方/星），不仅靠颜色 |
| 幽灵穿墙后蛇尾视觉"跳变" | 已接受；真实贪吃蛇传统穿墙即是这种体验 |
| 浏览器 autoplay 策略阻断 AudioContext | `useSound` 现有 `ctx.resume()` 已处理 |
| AudioContext 在 unmount 未释放 | 合成音是一次性 osc，用完自动 GC；无需 close |
| 合成音听感不如 mp3 | 已接受；以后可重构 composable 改 mp3 |
